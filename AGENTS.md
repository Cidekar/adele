## Orientation

You are inside an app generated by `adele new <name>`. Top-level layout:

```
config/cors.yml
handlers/handlers.go
middleware/{middleware,nosurf,trustedProxy}.go
models/models.go
resources/views/{home,404,layouts/base}.jet
main.go
routes.go            ← mounts probes + static + Web/Api groups
routes-web.go        ← web routes (Jet-rendered)
routes-api.go        ← API routes
types.go             ← application{} wrapper
go.mod
.env                 ← APP_KEY pre-generated, DB/MAIL keys stubbed
```

There is **no `cmd/` directory**. `main.go` is the entry point. There is **no `internal/`**, no `db/`, no `app/`. Do not invent these.

**File placement — non-negotiable. Put things here, do not ask, do not invent paths:**

| What | Where |
|------|-------|
| Jet view templates  | `resources/views/<name>.jet` |
| Layouts/partials    | `resources/views/layouts/`, `resources/views/partials/` |
| Mail templates      | `resources/mail/` |
| Handlers            | `handlers/<thing>.go` |
| Models              | `models/<thing>.go` |
| Middleware          | `middleware/<thing>.go` |
| Web routes          | `routes-web.go` |
| API routes          | `routes-api.go` |
| Config (YAML)       | `config/` |
| Static/build assets | `public/` |

If a file doesn't fit a row above, it does not get a new directory — extend an existing one. No `cmd/`, `internal/`, `db/`, `app/`, or `templates/`.

The framework owns: HTTP server, routing (`*mux.Mux` wrapping `*chi.Mux`), middleware base chain (RequestID, RealIP, RateLimiter, CORS, Recoverer, SessionLoad, Maintenance), DB pool, sessions, render (Jet/Go templates/Inertia), mailer, cache (Redis or Badger), RPC server, scheduler. The app owns: handlers, route declarations, business logic, optional wrappers.

## Bootstrap

`main.go` contains `bootstrapApplication()`. Read it before changing anything.

```go
func bootstrapApplication() *application {
    path, _ := os.Getwd()
    a := &adele.Adele{}
    a.New(path)                                     // framework wires .env, sessions, mux, DB, render, scheduler
    a.AppName = "<your-app-name>"

    app := &application{
        App:        a,
        Handlers:   &handlers.Handlers{App: a},
        Mail:       &a.Mail,
        Middleware: &middleware.Middleware{App: a},
    }
    app.App.Routes = app.routes()
    app.Models = models.New(a)                      // populates models.DB package-level var

    a.Provider = &provider.Provider{
        EnabledProviders: map[string]bool{},
        ProviderConfigs:  map[string]map[string]interface{}{},
    }
    a.Provider.LoadProviders(app.App)               // calls Register on every blank-imported provider
    return app
}
```

Rules:

- **MUST** call `a.New(path)` before constructing `application{}`. The framework populates `a.Routes`, `a.DB`, `a.Render`, `a.Session` during `New`. Reading these earlier yields nil.
- **MUST** assign `app.App.Routes = app.routes()` *after* `bootstrapApplication` constructs the wrapper. The framework already populated `a.Routes` during `New`; `app.routes()` adds your declarations on top.
- **MUST** call `a.Provider.LoadProviders(app.App)` *after* the provider list is constructed. Blank-imported providers register themselves at package init; LoadProviders calls their `Register` method in priority order.
- **NEVER** construct `*adele.Adele` by hand for production (`&adele.Adele{Log: logrus.New()}` is a test-only shape). Always go through `a.New(path)`.

## Routing

The skeleton ships three files. They mount onto `a.App.Routes` (the framework's `*mux.Mux`).

`routes.go` — top-level mounts:

```go
func (a *application) routes() *mux.Mux {
    a.App.Routes.Method("Get", "/public/*", http.StripPrefix("/public", secureFileServer))
    a.App.Routes.Mount("/", a.WebRoutes())
    a.App.Routes.Mount("/api", a.ApiRoutes())
    return a.App.Routes
}
```

`routes-web.go` — Jet-rendered routes, NoSurf-protected:

```go
func (a *application) WebRoutes() http.Handler {
    r := mux.NewRouter()
    r.Use(a.Middleware.NoSurf)
    r.NotFound(a.Handlers.NotFound)
    r.Get("/", a.Handlers.Home)
    return r
}
```

`routes-api.go` — same shape, `/api`-mounted, no NoSurf.

Rules:

- **MUST** declare routes via `a.App.Routes.Get/Post/...` or `mux.NewRouter()`. Both call into the framework's scope-annotation parser, which strips `[scopes:...]` annotations and registers them. Bypassing `mux.Mux` (e.g. `chi.NewRouter()` directly at top level) silently breaks the OAuth2 scope system.
- **MAY** import `github.com/go-chi/chi/v5` inside group closures (`r.Route("/foo", func(mux chi.Router) {...})`). Chi is a direct dep in the skeleton. Use it for `chi.URLParam`, `chi.URLParamFromCtx` when you need parameter access.
- **MUST** mount operator probes (`/healthz`, `/readyz`, `/metrics`) at the root via `a.App.Routes.Get(...)`, NOT inside WebRoutes/ApiRoutes. Probes need to bypass the per-router middleware tier (auth, rate limit, structured logging).
- **NEVER** roll your own RateLimit, RequestID, RealIP, or Recoverer middleware. The framework's `BootstrapMux` mounts these on the parent mux during `a.New(path)`. Re-implementing them creates a chain that runs each twice.
- **NEVER** stamp `r.RemoteAddr` into a typed context key as a "bridge." Chi's RealIP (mounted by the framework) already rewrites `r.RemoteAddr` in place. Read it directly with `net.SplitHostPort(r.RemoteAddr)`.

`adele install starter-kit --with-auth` rewrites `routes-web.go`. If you have hand-edits there, copy them out first.

## Handlers

Handlers are methods on `*Handlers` in package `handlers`, with `App *adele.Adele` injected:

```go
type Handlers struct {
    App    *adele.Adele
    Models *models.Models
}

func (h *Handlers) Home(w http.ResponseWriter, r *http.Request) {
    err := h.App.Helpers.Render(w, r, "home", nil, nil)
    if err != nil {
        h.App.Log.Error("error rendering:", err)
    }
}
```

Rules:

- **MUST** use `h.App.Helpers.Render(w, r, view, variables, data)` for Jet rendering. Variables are template-scope; data is passed to the layout.
- **MUST** read JSON bodies with `h.App.Helpers.ReadJSON` and validate with `h.App.Helpers.NewValidator(values)`. Do not hand-roll a JSON decoder.
- **MUST** log via `h.App.Log` (logrus). Do not import logrus directly.
- **NEVER** add bare functions taking `*adele.Adele`. The skeleton's convention is methods on `*Handlers`. Routing passes `a.Handlers.Foo`, not `handlers.Foo(a)`.
- **NEVER** invent context keys for cross-handler state without checking framework imports first. The skeleton ships zero context keys. If you need one, name it (`type ctxKey string`) and document it next to the writer.

## Models / Database

The skeleton's data layer is one file:

```go
// models/models.go
package models
import "github.com/upper/db/v4"

var DB upper.Session
type Models struct{}

func New(a *adele.Adele) *Models {
    DB = a.DB.NewSession()
    return &Models{}
}
```

Rules:

- **MUST** use the framework's session: `a.DB.NewSession()` returns a fresh `upper.Session`. The raw `*sql.DB` is at `a.DB.Pool` if you need it (migration tooling, `/readyz` pingers).
- **MUST** call `models.New(a)` after `app.routes()` in `bootstrapApplication` so `models.DB` is populated before the first request.
- **NEVER** construct `&database.Database{Pool: sqlDB}` by hand. The framework owns this; instances reach it through `a.DB`.
- **NEVER** add a wrapper package (`db/`, `repo/`, etc.) prematurely. Add a `models/<thing>.go` file with the type and methods. If a real second persistence target appears later, factor at that point.

The skeleton ships `models.DB` as a package-level var, not injected through `Models`. That's the convention. Don't refactor it on first sight.

## Configuration

`config/cors.yml` is the only config file the skeleton ships. Most configuration is via environment variables (read with `a.Helpers.Getenv(key, default)`):

| Var | Purpose |
|---|---|
| `APP_NAME`, `APP_KEY`, `APP_URL`, `APP_DEBUG` | Framework |
| `DATABASE_TYPE` (`postgres` / `mysql`), `DATABASE_HOST/PORT/USER/PASSWORD/NAME`, `DATABASE_SSL_MODE` | DB connection |
| `COOKIE_SECURE`, `COOKIE_DOMAIN` | NoSurf middleware |
| `TRUSTED_PROXIES`, `TRUST_PROXY_HEADERS` | Trusted-proxy middleware (not wired by default) |
| `MAILER_*`, `SMTP_*` | Mailer |
| `RPC_SERVER_PORT` | Framework RPC server |

Rules:

- **MUST** read env via `a.Helpers.Getenv(key, default)`. Do not call `os.Getenv` directly — the helper handles the .env load order.
- **MUST** use `strconv.ParseBool` / `Atoi` / `time.ParseDuration` inline when typed values are needed. The skeleton has no `pkg/config` wrapper.
- **NEVER** check `DATABASE_SSL_MODE=prefer` against `lib/pq` — it doesn't accept that mode. Use `pgx5` driver in the migrate URL or set `sslmode=disable`/`require`.

## Adding routes

```go
// routes-api.go
func (a *application) ApiRoutes() http.Handler {
    r := mux.NewRouter()
    r.Get("/health", a.Handlers.HealthStatus)
    r.Post("/users[scopes:user-create]", a.Handlers.CreateUser)
    return r
}
```

The `[scopes:...]` annotation is parsed by `mux.Mux.Get/Post/...` and registered for the OAuth2 scope-checking middleware. Multiple scopes are space-separated: `[scopes:read write]`.

## Adding the queue (`adele-queue`)

Blank-import to register the provider:

```go
import _ "github.com/cidekar/adele-queue"
```

Configure before `LoadProviders`:

```go
a.Provider.SetProviderConfig("queue", map[string]interface{}{
    "backend":               "redis",   // "memory" or "redis"
    "worker_count":          4,
    "max_attempts":          5,
    "queue_channels":        []string{"default", "email"},
    "queue_channel_default": "default",
    "redis_prefix":          "<your-app>",
})
```

Get the queue handle (provider name is `"queue"`, type assertion is `*queue.ServiceProvider`):

```go
for _, sp := range provider.GetRegisteredProviders() {
    if sp.Name() == "queue" {
        if qp, ok := sp.(*queue.ServiceProvider); ok {
            q := qp.Service()      // *queue.Queue
            // q.RegisterHandler / q.RegisterHandlerCtx / q.Dispatch / q.DispatchIn / q.Depth
        }
    }
}
```

Register a handler and dispatch:

```go
q.RegisterHandlerCtx("send-email", func(ctx context.Context, payload interface{}) error {
    bytes, _ := payload.([]byte)
    var p MyPayload
    json.Unmarshal(bytes, &p)
    // ... send email
    return nil
})

bytes, _ := json.Marshal(MyPayload{To: "x@example.com"})
q.Dispatch(queue.Job{Name: "send-email", Payload: bytes, Retry: true, RetryInSeconds: 5})
```

Rules:

- **MUST** pin `adele-queue` to v1.0.2 or later. Earlier versions don't expose `Service()` accessor on the provider.
- **MUST** configure `a.Cache` as a Redis cache before setting `backend: redis` on the queue. The queue takes its Redis pool from `a.Cache` — it does not configure Redis itself. If `a.Cache` isn't a `*redisdriver.RedisCache`, redis dispatches silently fail.
- **NEVER** copy the README's `q := queue.New(app)` line as-is. The actual signature is `func New(*adele.Adele) (*Queue, error)` — it returns `(*Queue, error)`. The README example won't compile.
- **NEVER** trust the queue docs' RPC-server claim. The current source dispatches handlers in-process via the registry; `RPC_SERVER_PORT` and `RPC_PORT` env vars are NOT consumed by the queue (despite docs claiming otherwise).
- **MAY** use `Job.RetryInSeconds` for linear backoff, or `q.DispatchIn(job, delay)` to schedule a job no sooner than `delay` from now (useful for honoring upstream `Retry-After` hints).

The queue ships **no MySQL migration**. Postgres only.

## Adding OAuth2 (`adele-oauth2`)

Blank-import:

```go
import _ "github.com/cidekar/adele-oauth2"
```

Provide `config/oauth.yml` (the framework auto-seeds the embedded default if missing):

```yaml
GuardedRouteGroups:
  - /api
UnguardedRoutes:
  - /api/health
AuthorizationTokenTTL: 120s
OauthTokenTTL: 24h
RefreshTokenTokenTTL: 48h
Scopes:
  user-create: Permission to create users
  user-read: Permission to read users
```

Run the migrations (Postgres only — see Cross-cutting):

```sh
adele migrate up
# Apply BOTH migrations from the adele-oauth2 module:
#   migrations/oauth_tables.postgres.sql
#   migrations/add_flow_column.sql        ← REQUIRED, adds oauth_clients.flow
```

Mount the bearer middleware on guarded routes:

```go
// In routes-api.go
oauthSvc := lookupOAuthService(a)   // walks provider.GetRegisteredProviders()
r.Use(oauthSvc.AuthenticationTokenMiddleware())
r.Use(oauthSvc.AuthenticationCheckForScopes())
r.Post("/api/users[scopes:user-create]", a.Handlers.CreateUser)
```

```go
func lookupOAuthService(a *adele.Adele) *adeleoauth2api.Service {
    for _, sp := range provider.GetRegisteredProviders() {
        if sp.Name() == "oauth" {
            if op, ok := sp.(*adeleoauth2.ServiceProvider); ok {
                return op.Service()
            }
        }
    }
    return nil
}
```

After token validation the middleware stamps three context keys:

```go
api.ContextKeyClientID    // int — oauth_clients.id
api.ContextKeyClientName  // string — oauth_clients.name
api.ContextKeyAccessToken // string — token plaintext (legacy string key "accessToken" also stamped)
```

Read with `r.Context().Value(api.ContextKeyClientName).(string)`.

Rules:

- **MUST** pin `adele-oauth2` to **v1.0.5 or later**. Earlier versions do not expose `(*ServiceProvider).Service()`, which is the only sanctioned way to mount the bearer middleware on a non-default subrouter.
- **MUST** set `GuardedRouteGroups` in oauth.yml or via `SetProviderConfig`. **An empty `GuardedRouteGroups` with the bearer middleware mounted returns HTTP 500 on every request, including healthchecks.** This is the most common bootstrap trap.
- **MUST** apply BOTH the main migration (`oauth_tables.postgres.sql`) AND `add_flow_column.sql`. Without `flow`, every `authorization_code` client silently fails the flow check and returns `unsupported_grant_type`.
- **MUST** use Postgres. The MySQL migration is broken as shipped (declares `bytea`, a Postgres type; declares `token NOT NULL` columns the Go code doesn't write to). `DATABASE_TYPE=mysql` with adele-oauth2 will not work without manual schema fixes.
- **MUST** match scope keys against the regex `^[a-zA-Z0-9-]+$`. Underscores and colons are rejected at config-load — the framework `panic`s on `New`. So `read:write` and `user_create` both crash.
- **NEVER** assume `GuardedRouteGroups` is prefix-matched. The check uses `strings.Contains(path, group)`, so `["/api"]` also guards `/private/api/foo` and `/foo/api/bar`. If you want strict prefix semantics, fork the middleware.
- **NEVER** read the typed `api.ContextKeyAccessToken` from your own scope-checking code — the framework's `ScopeHandler` reads the legacy string key `"accessToken"`. Both are stamped today; the legacy key is the canonical read path.
- **NEVER** confuse the OAuth provider name. It's `"oauth"`, not `"oauth2"`. `provider.GetRegisteredProviders()` returns providers by `Name()`; `"oauth2"` will never match.

The `/oauth/token`, `/oauth/token/refresh`, `/oauth/authorize` (GET + POST), and `/api/ping` routes are auto-mounted by `Register`. You do not declare them yourself.

## Starter kit (`adele install starter-kit`)

The verb is `adele install starter-kit`, not `adele kit install`.

```sh
adele install starter-kit                    # vanilla — Vite + TS + Tailwind
adele install starter-kit --vue              # Vue 3 + Vite
adele install starter-kit --vue=2            # Vue 2.7 (legacy)
adele install starter-kit --no-tailwind      # skip Tailwind
adele install starter-kit --with-auth        # adds form-based auth scaffolding (aerra)
adele install starter-kit --vue --with-auth  # Vue 3 SPA + auth
```

Rules:

- **MUST** confirm with the literal word `yes` (case-insensitive) at the destructive prompt. A bare `y` cancels.
- **MUST** run `go mod tidy` after install — the kit drops new files with new imports.
- **MUST** run `make migrate:up` if installed with `--with-auth`. Without migrations, `/registration` and `/login` return 500 with `relation "users" does not exist`.
- **NEVER** run `--with-auth` if you've hand-edited `routes-web.go`, `main.go`, or `Makefile`. The install rewrites all three. The destructive prompt lists them but the surprise is that an "auth scaffold" replaces your entry point. Copy edits out first.
- **NEVER** combine `--vue=2` with `--with-auth`. The CLI rejects this combination — Vue 2 is end-of-life and the auth views target Vue 3.

The starter-kit does NOT install OAuth2. Form-based session auth and OAuth2 are separate paths. For OAuth2, follow the section above.

## Testing

```sh
go test ./...                      # all packages
go test -race ./...                # with race detector
go vet ./...
```

The skeleton ships exactly one test (`middleware/trustedProxy_test.go`). There is no `Makefile` test target beyond `go build`. Add tests as you add code.

Rules:

- **MUST** run `-race` before declaring a feature done. The framework's middleware mutates request state across goroutines.
- **MAY** construct an in-memory `*adele.Adele` for unit tests: `&adele.Adele{Log: logrus.New(), Debug: true}`. Do not call `a.New(path)` in tests — it reads `.env` and binds DB.
- **MUST** mock the queue's `Dispatcher` interface and the OAuth2 service via small local interfaces in handler tests. Don't import `adele-queue` or `adele-oauth2` packages directly in unit tests.

## Common traps (the ones that burned the last agent)

1. **You re-implement framework middleware.** The framework's `BootstrapMux` mounts RequestID, RealIP, RateLimiter, CORS, Recoverer, SessionLoad, Maintenance on the parent mux. Don't add yours. If you need a per-route limiter on top, define it as a small `func(http.Handler) http.Handler` and `r.Use` it inside the group, NOT at the root.

2. **You build a wrapper package around `*adele.Adele`.** Type aliases re-exporting framework symbols inevitably drift. Always import the framework directly (`adele "github.com/cidekar/adele-framework"`).

3. **You add `internal/`.** The skeleton has no `internal/`. The Go package boundary it enforces is rarely worth the layout cost in a single-binary app.

4. **You build speculative repos and helpers.** The skeleton's `models/models.go` is 11 lines. Add a model when you need a model. `WithTx`, `Repository[T]`, generic Result types — wait until a second consumer appears.

5. **You read framework docs through prior wrapper assumptions.** When in doubt, `go doc github.com/cidekar/adele-framework` against the version in go.mod, or read the source in `$(go env GOMODCACHE)/github.com/cidekar/adele-framework@<version>/`. Don't guess.

6. **You delete "dead" code via grep without checking reference-style usage.** Functions called via slices (`Middleware(a)` returning a `[]func`) or method values (`r.Use(handler.X)`) won't show up in a `pkg.Func(` grep. Use `staticcheck -checks=U1000` or a multi-pattern search. Better: leave it alone unless you have a reason to delete.

7. **You bridge `r.RemoteAddr` into a context key.** Chi's RealIP (framework-mounted) has already rewritten `r.RemoteAddr`. Read it directly. Adding a "typed key" middleware on top is a tax with no benefit.

## When you don't know what to do

In priority order — earlier sources are auto-generated from source code and cannot drift; later sources are interpretation that can.

1. **Look up the symbol on pkg.go.dev** (lowest friction, ground truth):
   - https://pkg.go.dev/github.com/cidekar/adele-framework
   - https://pkg.go.dev/github.com/cidekar/adele-queue
   - https://pkg.go.dev/github.com/cidekar/adele-oauth2

   Append `@<version>` to pin to your `go.mod` version, e.g.
   `https://pkg.go.dev/github.com/cidekar/adele-framework@v1.0.3`.
   The default URL serves the latest tagged version, which may be ahead of what your app pins.

2. **`go doc` locally** (offline equivalent — same content):
   ```sh
   go doc github.com/cidekar/adele-framework Adele
   go doc github.com/cidekar/adele-queue Queue.Dispatch
   go doc github.com/cidekar/adele-oauth2/api Service.AuthenticationTokenMiddleware
   ```
   Works against whatever's in `$GOPATH/pkg/mod/` after `go mod tidy`. No network required.

3. **Read the skeleton file you're about to edit.** The comments are sparse but accurate.

4. **Read [`adele-package.md`](https://github.com/Cidekar/adele-documentation/blob/main/skills/adele-package.md)** for opinionated wiring guidance, version pins, and known traps.

5. **Read the framework source** in `$(go env GOMODCACHE)/github.com/cidekar/adele-framework@<version>/` for behavior pkg.go.dev's signatures don't fully convey.

6. **If still uncertain, ask the human.** Do not invent a wrapper, do not invent a context key, do not invent a package.

> Caveat: pkg.go.dev only indexes packages that have been published to the Go module proxy. If you're testing against an unreleased branch, fall back to step 2 (`go doc` against a local clone) or step 5 (read the source directly).
